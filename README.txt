# Генератор случайных паролей

Программа для генерации случайных паролей с настройкой длины, алфавита и количества

## Сборка

cmd
mkdir build
cd build
cmake -S .. -G "Visual Studio 18 2026" -A x64
cmake --build . --config Debug

Исполняемый файл появится в build\app\Debug\my_program.exe

## Использование
my_program.exe [опции]

## Основные опции
-minl N минимальная длина пароля
-maxl N максимальная длина пароля
-n N точная длина (несовместима с -minl/-maxl)
-c N количество паролей (по умолчанию 1)
-a ALPH пользовательский алфавит (несовместим с -C)
-C CHARS наборы символов: a (строчные), A (заглавные), D (цифры), S (спецсимволы)
-d CHAR добавить разделитель для значений опций
-D CHAR заменить стандартные разделители (=, :) на CHAR
-help показать справку

## Формат передачи аргументов
Опции можно передавать слитно, через пробел или через разделитель. Опции могут идти в произвольном порядке.
Примеры корректного синтаксиса:
-minl10
-minl 10
-minl=10
-minl:10
-d! -minl!10

ПРИМЕРЫ
5 паролей длиной от 8 до 12 символов из строчных букв и цифр
my_program.exe -minl 8 -maxl 12 -c 5 -C aD

3 пароля длиной ровно 10 символов из пользовательского алфавита
my_program.exe -n 10 -a abc123 -c 3

## Использование кастомного разделителя !
my_program.exe -d! -minl!8 -maxl!12 -c!5 -C!aD

## Валидация и обработка ошибок
Программа проверяет корректность аргументов: 
	дублирование опций — одна и та же опция не может быть указана дважды
	несовместимость опций: -n и -minl/-maxl несовместимы; -a и -C несовместимы; -m1/-m2 и -n несовместимы

## Специальные опции:
Если указана -m1, обязательно должна быть -m2
В опции -C нельзя дублировать символы (например, -C aDa — ошибка)
Неизвестные опции (например, -s, -i, -m3) игнорируются
В случае ошибки программа выводит сообщение и завершает работу с ненулевым кодом возврата

## Структура проекта


Корневая директория:
-'CMakeLists.txt' 
- 'README.txt' 

Директория "app/src":
- 'main.c' 

Директория "lib/src":
- 'lib_main.c' 

Директория "lib/include":
- 'lib_main.h' 

## Проект разделен на два модуля:
Основная программа (app/) — осуществляет вызов библиотечных функций
Библиотека (lib/) — содержит все вспомогательные функции
Библиотека может быть собрана как статическая (по умолчанию) или динамическая (с опцией -DBUILD_SHARED_LIBS=ON).
